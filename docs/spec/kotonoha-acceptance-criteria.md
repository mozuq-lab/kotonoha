# 文字盤コミュニケーション支援アプリ（kotonoha） 受け入れ基準

## 概要

このドキュメントは文字盤コミュニケーション支援アプリ（kotonoha）の受け入れ基準とテスト項目を記載します。

## 機能テスト基準

**【信頼性レベル凡例】**:
- 🔵 **青信号**: EARS要件定義書・設計文書・ユーザヒアリングを参考にした確実なテスト基準
- 🟡 **黄信号**: EARS要件定義書・設計文書・ユーザヒアリングから妥当な推測によるテスト基準
- 🔴 **赤信号**: EARS要件定義書・設計文書・ユーザヒアリングにない推測によるテスト基準

### REQ-001: 五十音文字盤の表示 🔵 *spec_original.md 5-1-1*

**Given（前提条件）**:
- アプリが起動している
- メイン画面が表示されている

**When（実行条件）**:
- ユーザーがメイン画面を見る

**Then（期待結果）**:
- 五十音配列の文字盤が画面に表示される
- すべての文字（あ〜ん、濁音、半濁音、拗音）が表示される
- 各文字ボタンのサイズが44px × 44px以上である
- 文字が読みやすいフォントサイズで表示される

**テストケース**:
- [ ] 正常系: 起動直後に五十音文字盤が正しく表示される
- [ ] 正常系: 画面回転（縦・横）でも文字盤が適切に表示される
- [ ] 正常系: フォントサイズ変更時に文字盤の文字も追従する
- [ ] 境界値: 最小フォントサイズでも文字が読める
- [ ] 境界値: 最大フォントサイズでも文字盤が画面内に収まる

### REQ-002: 文字盤タップで文字入力 🔵 *spec_original.md 5-1-1*

**Given（前提条件）**:
- 文字盤が表示されている

**Then（期待結果）**:

- 視覚的フィードバック（タップアニメーション等）が表示される

**テストケース**:
- [ ] 正常系: 1文字タップで正しく入力欄に反映される
- [ ] 正常系: 連続で複数文字をタップして文章を作成できる
- [ ] 正常系: 濁音、半濁音、拗音も正しく入力される
- [ ] 異常系: 連続タップ（ダブルタップ）で文字が重複入力されない
- [ ] 性能: タップから100ms以内に文字が反映される

### REQ-301: 緊急ボタンの2段階確認 🔵 *spec_original.md 5-1-4、ユーザヒアリング*

**Given（前提条件）**:
- アプリがどの画面でも表示されている
- 緊急ボタンが常に表示されている

**When（実行条件）**:
- ユーザーが緊急ボタンを1回タップする

**Then（期待結果）**:
- 確認ダイアログ「緊急呼び出しを実行しますか?」が表示される
- ダイアログに「はい」「いいえ」ボタンが表示される
- 「いいえ」をタップするとダイアログが閉じる
- 「はい」をタップすると緊急音が鳴り、画面が緊急表示に切り替わる

**テストケース**:
- [ ] 正常系: 緊急ボタン1回目タップで確認ダイアログが表示される
- [ ] 正常系: 「はい」タップで緊急音が鳴り、画面が赤く表示される
- [ ] 正常系: 「いいえ」タップでダイアログが閉じ、何も起きない
- [ ] 異常系: 確認ダイアログ外をタップしてもダイアログが閉じない（誤操作防止）
- [ ] 異常系: 緊急音が鳴っている間、他の操作を受け付けない（意図的な設計かを確認）

### REQ-401: TTS読み上げ機能 🔵 *spec_original.md 5-2-1、ユーザヒアリング*

**Given（前提条件）**:
- 入力欄に「こんにちは」と入力されている
- OS標準TTSが利用可能な環境

**When（実行条件）**:
- ユーザーが読み上げボタンをタップする

**Then（期待結果）**:
- 「こんにちは」とOS標準TTSで読み上げられる
- 読み上げ開始までの時間が1秒以内
- 読み上げ中は読み上げボタンが「停止」ボタンに変わる
- 停止ボタンをタップすると読み上げが中断される

**テストケース**:
- [ ] 正常系: 入力欄のテキストが正しく読み上げられる
- [ ] 正常系: 定型文タップ時も即座に読み上げられる
- [ ] 正常系: 大ボタンタップ時も即座に読み上げられる
- [ ] 正常系: 読み上げ中に停止ボタンで中断できる
- [ ] 正常系: 読み上げ速度（遅い/普通/速い）が設定通りに反映される
- [ ] 異常系: 入力欄が空の場合、読み上げボタンが無効化される
- [ ] 異常系: OS音量が0の場合、「音量が0です」という警告が表示される
- [ ] 性能: 読み上げボタンタップから1秒以内に読み上げが始まる

### REQ-901: AI変換機能（短い入力→丁寧な文章） 🔵 *spec_original.md 5-6-1、ユーザヒアリング*

**Given（前提条件）**:
- アプリがオンライン状態
- 入力欄に「水 ぬるく」と入力されている
- AI丁寧さレベルが「普通」に設定されている

**When（実行条件）**:
- 入力完了後、自動的にAI変換が開始される

**Then（期待結果）**:
- AI変換候補「お水をぬるめでお願いします」が表示される
- 変換結果を確認できる画面が表示される
- 「採用」「再生成」「元の文を使う」ボタンが表示される
- 「採用」をタップすると、変換結果が入力欄に反映される
- 「元の文を使う」をタップすると、「水 ぬるく」がそのまま使われる

**テストケース**:
- [ ] 正常系: 短い入力が自然な文章に変換される
- [ ] 正常系: 変換結果を採用すると入力欄に反映される
- [ ] 正常系: 元の文を使うと変換前の文が使われる
- [ ] 正常系: 再生成をタップすると別の変換候補が表示される
- [ ] 正常系: 丁寧さレベル「カジュアル」「普通」「丁寧」で変換結果が変わる
- [ ] 異常系: オフライン時は「AI変換が利用できません」と表示される
- [ ] 異常系: オフライン時は「元の文を使う」ボタンのみ表示される
- [ ] 異常系: AI変換APIエラー時は元の文を使うフォールバック処理が実行される
- [ ] 性能: 変換応答時間が平均3秒以内
- [ ] 性能: 3秒を超える場合はローディング表示が出る

---
